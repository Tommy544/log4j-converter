Cieľom nášho projektu bolo vytvoriť prevádzač log4j konfigurácie z formátu XML do properties a naspäť s pomocou Javy, dopytovacích jazykov a rozhraniami pre prácu s XML dokumentami.

 

 

 

Moja práca spočívala v návrhu aplikačného kódu, t. j. vytvoreniu tried a metód, z ktorými program pracuje pri konverzií formátov (viď obrázok) . Taktiež som implementoval verejné metódy.

 

 

 

 

Pri konverzií z xml formátu do properties sa ako najvýhodnejšie javilo použitie XSL transformácií (XSLT). Java štandardne obsahuje prostriedky pre vykonávanie XSLT. Vstupný XML súbor je skontrolovaný pomocou DTD. Výstup properties formátu môže byť na terminál (štandardný výstup) alebo do textového súboru.

 

 

Pri konverzií z properties do XML sme použili DOM (Document Object Model) API štandardne nachádzajúce sa v Jave. Taktiež je podporovaný výstup do XML súboru alebo na štandardný výstup.
